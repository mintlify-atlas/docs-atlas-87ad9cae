lazydocker is a simple terminal UI for Docker and Docker Compose, written in Go. Instead of juggling multiple terminal windows and memorizing command syntax, you get a single interface where every common operation is one keypress away — with live log streaming, ASCII metrics graphs, and full mouse support.

## Why lazydocker

Managing containers by hand means a constant context-switch between `docker ps`, `docker-compose logs --follow`, `docker-compose restart`, and everything in between. If a service dies right after you restart it, you only find out by polling. If you want logs for just one service, you lose that terminal window for as long as you care about it.

lazydocker solves this by putting the state of your entire Docker environment — running containers, service health, logs, CPU and memory graphs — into one persistent terminal panel. You can restart a container, tail its logs, exec into a shell, and prune unused images without ever leaving the UI or recalling flag syntax.

## Requirements

- Docker >= **29.0.0** (API >= **1.24**)
- Docker Compose >= **1.23.2** (optional — only required for Compose workflows)
