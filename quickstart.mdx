Once lazydocker is installed, you can have your entire Docker environment visible and interactive in under a minute. This guide walks you through launching the UI, setting up a convenience alias, and learning the essential keybindings for daily use.

## Launch lazydocker

<Steps>
  <Step title="Start lazydocker">
    Run the following command in any terminal:

    ```bash
    lazydocker
    ```

    lazydocker reads your current directory for a `docker-compose.yml` file and connects to the Docker daemon automatically. The UI opens immediately â€” no configuration required.
  </Step>

  <Step title="Create a short alias">
    Typing `lazydocker` repeatedly gets old fast. Add an alias to your shell config:

    ```bash
    echo "alias lzd='lazydocker'" >> ~/.zshrc
    ```

    Substitute `.zshrc` for `.bashrc`, `.bash_profile`, or whichever file your shell sources on startup. Reload the config:

    ```bash
    source ~/.zshrc
    ```

    From now on, `lzd` launches the UI.
  </Step>

  <Step title="Navigate the panels">
    The UI is divided into side panels on the left and a main detail panel on the right. Use the number keys to jump directly to any panel:

    | Key | Panel |
    |-----|-------|
    | `1` | Project |
    | `2` | Services |
    | `3` | Containers |
    | `4` | Images |
    | `5` | Volumes |
    | `6` | Networks |

    Within a panel, use the arrow keys or `j`/`k` to move between items. Press `Enter` to focus the main panel. Press `Esc` to return to the side panel. Use `[` and `]` to cycle through tabs in the main panel (logs, stats, config, and more).
  </Step>

  <Step title="Perform your first operations">
    With a container selected in the Containers panel (`3`), try these keybindings:

    | Key | Action |
    |-----|--------|
    | `m` | View logs |
    | `r` | Restart container |
    | `s` | Stop container |
    | `p` | Pause / unpause |
    | `a` | Attach to container |
    | `E` | Exec a shell inside the container |
    | `d` | Remove container |
    | `e` | Hide / show stopped containers |

    The same `m`, `r`, `a`, and `E` bindings work in the Services panel (`2`) for Docker Compose services.

    <Tip>
      Press `+` to cycle through screen modes: normal, half-screen, and fullscreen. Use `_` to go back. Fullscreen mode gives the main panel the entire terminal, which is useful for reading long log output.
    </Tip>
  </Step>
</Steps>

## CLI flags

lazydocker accepts several flags that you can pass at startup.

| Flag | Short | Default | Description |
|------|-------|---------|-------------|
| `--config` | `-c` | `false` | Print the current default configuration as YAML and exit. |
| `--debug` | `-d` | `false` | Enable debug logging. |
| `--file` | `-f` | _(none)_ | Specify one or more alternate Docker Compose files. Repeatable. |
| `--project` | `-p` | _(none)_ | Set the Docker Compose project name. |

### Print the default config

Use `-c` to see lazydocker's full default configuration. This is useful as a starting point for your own `config.yml`:

```bash
lazydocker -c
```

Example output (truncated):

```yaml
gui:
  scrollHeight: 2
  language: auto
  border: rounded
  theme:
    activeBorderColor:
      - green
      - bold
    inactiveBorderColor:
      - white
    selectedLineBgColor:
      - blue
  wrapMainPanel: true
  sidePanelWidth: 0.333
logs:
  timestamps: false
  since: 60m
  tail: ""
```

Redirect it to a file to create your user config:

```bash
lazydocker -c > ~/.config/lazydocker/config.yml
```

### Use alternate Compose files

To load a specific Compose file instead of the default `docker-compose.yml`:

```bash
lazydocker -f docker-compose.prod.yml
```

Stack multiple `-f` flags to merge files, just as you would with `docker compose`:

```bash
lazydocker -f docker-compose.yml -f docker-compose.override.yml
```

### Set a project name

Override the Docker Compose project name (which defaults to the current directory name):

```bash
lazydocker -p myproject
```
