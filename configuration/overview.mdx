title: "Configuring lazydocker: file locations and structure"
description: "Learn where lazydocker stores its config file on each OS, how to open and edit it, and what every top-level configuration key controls."
---

lazydocker reads a single `config.yml` file on startup. You can customize nearly every aspect of the UI, logs, commands, and graphs by editing this file. Changes take effect only after you restart lazydocker.

## Config file location

<Tabs>
  <Tab title="macOS">
    ```
    ~/Library/Application Support/jesseduffield/lazydocker/config.yml
    ```
  </Tab>
  <Tab title="Linux">
    ```
    ~/.config/lazydocker/config.yml
    ```
  </Tab>
  <Tab title="Windows">
    ```
    C:\Users\<User>\AppData\Roaming\lazydocker\config.yml
    ```
  </Tab>
</Tabs>

<Note>
  If the environment variable `CONFIG_DIR` is set, lazydocker uses that path instead of the platform default.
</Note>

## Opening the config file from inside lazydocker

You don't need to leave the terminal UI to open your config. Navigate to the **project panel** (top-left), then:

- Press **`o`** to open the config file with your default editor.
- Press **`e`** to open it directly in Vim.

After saving your changes, restart lazydocker for them to take effect.

## Printing the default config

To see the full default configuration, run:

```bash
lazydocker --config
# or
lazydocker -c
```

This prints the complete default YAML to stdout, which is useful as a starting point or to check what a field's default value is.

## JSON Schema and IntelliSense

lazydocker ships a JSON Schema for `config.yml`. If your config file is in one of the standard paths above, the [YAML Red Hat extension](https://marketplace.visualstudio.com/items?itemName=redhat.vscode-yaml) for VS Code enables autocompletion and error-checking automatically.

If you store your config in a custom location, add this comment to the top of the file to enable IntelliSense manually:

```yaml
# yaml-language-server: $schema=https://json.schemastore.org/lazydocker.json
```

## Top-level configuration keys

| Key                | Type     | Description                                              |
| ------------------ | -------- | -------------------------------------------------------- |
| `gui`              | object   | Visual settings: colors, panels, scrolling, borders      |
| `logs`             | object   | Log display: timestamps, time range, line limit          |
| `commandTemplates` | object   | Docker Compose command strings used by lazydocker        |
| `customCommands`   | object   | User-defined commands accessible via `c`                 |
| `bulkCommands`     | object   | Commands that apply to all items in a panel              |
| `oS`               | object   | Platform-level open-file and open-link commands          |
| `stats`            | object   | Stats collection duration and graph definitions          |
| `replacements`     | object   | Image name prefix substitutions for the Images panel     |
| `confirmOnQuit`    | bool     | Prompt before quitting with `q` or `Esc`                 |
| `ignore`           | string[] | List items whose names contain these strings are hidden  |

<Warning>
  If you set a top-level key (for example `commandTemplates:`) with no child values, lazydocker discards all defaults for that section. Always include at least the fields you want to override.
</Warning>

## Full default config

```yaml
gui:
  scrollHeight: 2
  language: "auto" # one of 'auto' | 'en' | 'pl' | 'nl' | 'de' | 'tr'
  border: "rounded" # one of 'rounded' | 'single' | 'double' | 'hidden'
  theme:
    activeBorderColor:
      - green
      - bold
    inactiveBorderColor:
      - default
    selectedLineBgColor:
      - blue
    optionsTextColor:
      - blue
  returnImmediately: false
  wrapMainPanel: true
  sidePanelWidth: 0.333
  showBottomLine: true
  expandFocusedSidePanel: false
  screenMode: "normal" # one of 'normal' | 'half' | 'fullscreen'
  containerStatusHealthStyle: "long" # one of 'long' | 'short' | 'icon'
logs:
  timestamps: false
  since: '60m' # set to '' to show all logs
  tail: '' # set to 200 to show last 200 lines of logs
commandTemplates:
  dockerCompose: docker compose
  restartService: '{{ .DockerCompose }} restart {{ .Service.Name }}'
  up: '{{ .DockerCompose }} up -d'
  down: '{{ .DockerCompose }} down'
  downWithVolumes: '{{ .DockerCompose }} down --volumes'
  upService: '{{ .DockerCompose }} up -d {{ .Service.Name }}'
  startService: '{{ .DockerCompose }} start {{ .Service.Name }}'
  stopService: '{{ .DockerCompose }} stop {{ .Service.Name }}'
  serviceLogs: '{{ .DockerCompose }} logs --since=60m --follow {{ .Service.Name }}'
  viewServiceLogs: '{{ .DockerCompose }} logs --follow {{ .Service.Name }}'
  rebuildService: '{{ .DockerCompose }} up -d --build {{ .Service.Name }}'
  recreateService: '{{ .DockerCompose }} up -d --force-recreate {{ .Service.Name }}'
  allLogs: '{{ .DockerCompose }} logs --tail=300 --follow'
  viewAlLogs: '{{ .DockerCompose }} logs'
  dockerComposeConfig: '{{ .DockerCompose }} config'
  checkDockerComposeConfig: '{{ .DockerCompose }} config --quiet'
  serviceTop: '{{ .DockerCompose }} top {{ .Service.Name }}'
oS:
  openCommand: open {{filename}}
  openLinkCommand: open {{link}}
stats:
  graphs:
    - caption: CPU (%)
      statPath: DerivedStats.CPUPercentage
      color: cyan
    - caption: Memory (%)
      statPath: DerivedStats.MemoryPercentage
      color: green
```
